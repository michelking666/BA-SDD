# SDD PRD Validate — 验证PRD契合度

## 触发方式
用户输入 `/sdd:prd-validate`

## 执行步骤

### 1. 确定验证对象
询问用户：基于哪个变更/PRD 文件进行验证？

### 2. 读取对比基准
- PRD 文件：`llmwiki/prd/active/<变更名>-prd.md`
- 规格文件：`openspec/changes/approved/<变更名>/spec.md`
- 原型文件：`openspec/changes/approved/<变更名>/prototypes.html`

### 3. 执行验证检查

**3.1 需求覆盖检查**
逐条核对 spec.md 中的 FR / NFR / BR 是否在 PRD 中有对应描述：

| 规格编号 | 规格描述 | PRD 章节 | 状态 |
|----------|----------|----------|------|
| FR-001 | | 4.1.2 | ✅/❌ |

**3.2 一致性检查**
- 功能描述是否与规格一致（无遗漏、无矛盾）
- 业务规则是否完整转录
- 验收标准是否覆盖

**3.3 完整性检查**
- 是否有章节缺失
- 是否有空内容节（待填写）
- 数据模型描述是否完整

**3.4 可读性检查**
- 专业术语是否定义
- 是否适合目标受众阅读

### 4. 生成验证报告

```markdown
## PRD 验证报告：<变更名>

> 验证时间：<YYYY-MM-DD>
> 整体评分：<通过/有问题需修正/不通过>

### 需求覆盖率：<X/Y 条 (XX%)>
**未覆盖需求**：
- FR-XXX：<描述>

### 不一致项
- <章节>：PRD 写的是「...」，规格要求是「...」

### 完整性缺失
- <缺失内容>

### 修改建议
1. <具体修改项>
2.
```

### 5. 提供修正路径
- 如有问题：指出具体修改位置，用户确认后直接更新 PRD 文件
- 全部通过：告知可以导出为 Word 格式交付
