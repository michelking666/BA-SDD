# SDD New — 阶段1：创建变更

## 触发方式
用户输入 `/sdd:new`

## 前置检查
检查 `openspec/changes/active/` 下是否有未完成的变更。如有，提示用户先完成或归档当前变更，禁止同时开启多个变更。

## 执行步骤

### 1. 收集变更信息
向用户询问：
- **变更名称**（kebab-case，如 `user-auth-v2`）
- **变更目标**：这次变更要解决什么问题？
- **变更范围**：涉及哪些功能模块？
- **背景说明**：为什么需要这次变更？（可选）

### 2. 创建变更目录和文件

创建目录：`openspec/changes/active/<变更名>/`

创建以下文件：

**change.md** — 变更定义文档：
```markdown
# 变更：<变更名>

> 创建时间：<YYYY-MM-DD>
> 状态：active

## 变更目标
<用户提供的目标>

## 变更范围
<用户提供的范围>

## 背景说明
<用户提供的背景>

## 阶段状态
- [x] 阶段1：创建变更 — <YYYY-MM-DD>
- [ ] 阶段2：提案评审
- [ ] 阶段3：实施变更
- [ ] 阶段4：生成测试
- [ ] 阶段5：验证变更
- [ ] 阶段6：归档变更
```

**tasks.md** — 任务清单（初始为空模板）：
```markdown
# 任务清单：<变更名>

## 待办任务
（阶段2提案后生成具体任务）

## 进行中

## 已完成
```

### 3. 输出确认
告知用户：
- 已创建变更 `<变更名>`，路径：`openspec/changes/active/<变更名>/`
- 下一步：运行 `/sdd:propose` 进入阶段2（提案评审）
